# GamePrice Project Context for OpenCode

## Stack
- Laravel 12 + PHP 8.4
- Vue 3 + Inertia.js + Tailwind CSS
- PostgreSQL + Redis
- Vite

## Project Structure
- `/tmp/gameprice-repo/` - Project root
- Models: Game, Store, Product, PriceHistory, PriceAlert, Review, User
- Controllers: GameController, PriceAlertController, ReviewController, etc.
- Commands: ScrapeEneba, ImportEnebaJson, ImportCheapSharkJson, ScrapeAllPrices, CheckPriceAlerts, UpdateSteamGames, etc.

## Key Models

### Product
- Fields: id, game_id, store_id, current_price, original_price, discount_percentage, currency, platform, region, url, affiliate_url, is_available, is_real_price, updated_at
- Relationships: belongsTo Game, belongsTo Store

### Game
- Fields: id, title, slug, description, cover_image, release_date, developer, publisher, genres (JSON), platforms (JSON), metacritic_score, steam_app_id, is_active
- Relationships: hasMany Product, hasMany Review, hasMany PriceAlert

### Store
- Fields: id, name, slug, logo_url, website_url, is_active, is_official
- Relationships: hasMany Product

### PriceHistory
- Fields: id, product_id, price, currency, recorded_at
- Relationships: belongsTo Product
- Note: Migration exists but NOT actively used yet

### PriceAlert
- Fields: id, game_id, email, target_price, is_active, notified_at, created_at
- Relationships: belongsTo Game
- Note: Model+Migration+Controller+Command exist but UI for creating alerts is missing

## Important Conventions
- Use `#[Fillable([...])]` attribute for model fillable (PHP 8.4 style)
- Use type hints in all methods
- Routes use Inertia::render for Vue pages
- Console commands in app/Console/Commands/
- Services in app/Services/
- Vue pages in resources/js/Pages/
- Components in resources/js/Components/
- Database migrations in database/migrations/
